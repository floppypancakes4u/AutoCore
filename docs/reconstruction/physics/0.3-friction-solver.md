# Phase 0.3 — Havok Vehicle Friction Solver & Axle Force Aggregation

Reverse-engineered from `autoassault.exe` (Ghidra project `AA-decode`, image base `0x400000`, Havok 2.3).
Read-only RE; no DB edits. Target: bit-exact C# port.

## Functions

| Symbol | Addr | Role |
|---|---|---|
| `hkVehicleFramework_postTickApplyForces` | `0x64bc70` | Per-wheel force build → per-axle aggregation → calls friction solver → writeback |
| `hkVehicleFrictionSolver_solve` | `0x6c4450` | Per-axle coupled longitudinal+lateral friction constraint solve + friction-circle clamp |
| `VehicleAction_calcWheelTorque` | `0x598040` | (upstream) writes clamped `[0,1000]` per-wheel drive torque to `wheels+0x28[i]` |

**Key architectural fact:** AutoAssault runs the friction solver over **2 constraint points = 2 axles** (front/rear), *not* per-wheel. Every per-wheel quantity (drive torque, contact jacobian, normal dir) is aggregated/averaged into its axle bucket in `postTickApplyForces` before the solver runs. Both loops in the solver iterate exactly 2 times.

## Part 2 — Axle force aggregation (`postTickApplyForces` @ 0x64bc70)

`this = hkVehicleFramework (param_1)`, `param_2 = dt vector` (`*param_2 == dt`).
Notation: `wheels = *(param_1+0xc)` (hkDefaultWheels collection); `wheelCount = wheels+0xc`; `axleCount = wheels+0x64`; wheel structs live at `*(wheels+0x80) + i*0xC0` (stride `0xC0`).

### 2a. Per-wheel loop (`for i in 0..wheelCount`)

For each wheel `i`, `axle = *(*(param_1+0x28) ... )`? — the axle index is `*(*(wheels+0x58) + i*4)` (call it `ax`). Let `WHEEL = *(wheels+0x80) + i*0xC0`.

**Suspension / normal impulse (applied immediately, before the solver):**
```
suspForce = (fw+0x28 -> +0x34)[i]           // per-wheel suspension force magnitude
s         = dt * suspForce
F.xyzw    = WHEEL[+0x30 .. +0x3c] * s        // contact-normal basis (unit) × force
chassisRB.applyPointImpulse(F, WHEEL)        // vtbl slot +0x60 on *(fw+0x30 .. +0x3c)
// if the wheel's contact body B is a movable RB (type==2):
F2.xyzw   = -(collisionScale[i]) * F.xyzw     // collisionScale = (*(fw..+0x4c))[i]
B.applyPointImpulse(F2, WHEEL)                // vtbl +0x60
```

**Per-axle jacobian + drive aggregation** (`invN = 1.0 / axleWheelCount`, where
`axleWheelCount = (int)*(wheels+0x68 + ax*4)`):
```
invN = g_flOne / (float)axleWheelCount

// --- angular jacobian rows (contact ang. impulse arm), averaged over the axle ---
AX[ax].jAng[0..3] += WHEEL[+0x20 .. +0x2c] * invN      // afStack_2cc[ax*0x14 + 0..3]

// --- lateral direction row from contact frame cross-products (unit-normalized) ---
AX[ax].jLat[0..2] += normalize(cross terms of WHEEL[+0x30..], WHEEL[+0x60..]) * ...   // +5,+6,+4
AX[ax].jLat.w     += latDir.w                                                          // +8, +9

// --- DRIVE IMPULSE pack (the engine-torque input to the friction solver) ---
driveTorque = wheels+0x28[i]          // calcWheelTorque output, clamped [0,1000]
wheelScale  = WHEEL[+0x88]            // per-wheel drive-scale (e.g. drive-wheel weight / radius factor)
AX[ax].drivePack += driveTorque * wheelScale * invN     // acStack_280 + ax*0x50 - 0x14
```

**=> Axle drive impulse formula (matches docs):**
```
drivePack[axle] = Σ_{wheels on axle} ( wheels+0x28[i] * WHEEL[+0x88] ) / axleWheelCount
```
Other per-axle scalars accumulated the same way (`× invN`): friction-table entries
`(*(wheels+0x34))[i]` and `(*(wheels+0x40))[i]` → stored at `AX -0x10` / `-0xc` (these are the
μ0 / slope / μmax friction-limit params consumed by the solver, already carrying the
setup-time **RearWheelFrictionScalar** = `vehicleData+0x740` for rear wheels), plus `suspForce`
→ `AX-0x8`, and a term `(sumForce)/mass` → `AX-0x4`.
`AX[ax] in-contact` byte (`acStack_280[ax*0x50]`) is set if any wheel on the axle is grounded.

### 2b. Solver invocation

```c
fStack_3f8 = *param_2;          // dt
fStack_3f4 = param_2[1];        // dt (both components carry the substep timestep)
hkVehicleFrictionSolver_solve(
    &fStack_3f8,                // param_1: {dt, dt}  (scalar timestep pair)
    (int)framework + 0x1fc,     // param_2: per-axle friction SETUP block (stride 0x64)
    auStack_2d8,                // param_3: chassis + contact solver body data (jacobian/Minv)
    (int)framework + 0x2cc);    // param_4: OUTPUT impulse array (stride 0x1c = 7 floats / axle)
```

### 2c. Writeback (`for i in 0..wheelCount`)

```
OUT = framework+0x2cc + (*(wheels+0x58))[i] * 0x1c      // this wheel's axle output slot
if (WHEEL+0x80 (inContact) && WHEEL+0xa8):
     WHEEL+0x94 = OUT[2]      // longitudinal (drive/brake) impulse
else WHEEL+0x94 = 0
WHEEL+0x98 = OUT[3]
WHEEL+0xa0 = OUT[1]           // lateral impulse
WHEEL+0x9c = OUT[0]
```

## Part 1 — Friction solver (`hkVehicleFrictionSolver_solve` @ 0x6c4450)

Signature: `void solve(float *dt2 /*param_1*/, AxleSetup *setup /*param_2 = fw+0x1fc*/, ChassisBody *cb /*param_3*/, float *out /*param_4 = fw+0x2cc*/)`

- `dt2[0] = dt2[1] = dt` (substep timestep; used as the impulse×time scale on friction limits).
- `setup` per-axle stride `0x64` (`0x19` floats), base `setup+0x54` for axle 0.
- `cb` = solver representation of the **chassis rigid body**: `cb+0xe0..0xec` = inverse-mass diagonal (linear), `cb+0x100..0x128` = inverse-inertia tensor (world), `cb+0xf0..` = center of mass, `cb+0xc0..0xcc` = accumulated linear impulse, `cb+0xd0..0xdc` = accumulated angular impulse, `cb+0x30` = pointer to the paired body (ground/other RB) solver record, `cb+0xbc` / `body+0xc` = "is fixed / infinite mass" flags. Each per-axle jacobian at `cb+0x20 + ax*0x50`.
- `out` per-axle stride `0x1c` (7 floats).

The solver runs in **four phases**, each looping the 2 axles.

### Phase A — Effective mass (Jacobian · M⁻¹ · Jᵀ), loop ×2

For each axle it builds two constraint rows (longitudinal drive row and normal/second row) as
`r × dir` arms against **both** bodies, transforms by each body's inverse inertia (skipping the
transform when the body's fixed-flag `cVar1`/`cVar2` is set → treated as infinite mass), and
forms the scalar effective mass:

```
Keff = Σ_axis ( jLin_a² * Minv_lin_a  +  jAng_a² * Iinv_a )      // over chassis body A
     + Σ_axis ( ... body B ... )
     + softness                                                   // = setup+0x44 + setup+0x94-ish
invKeff        = 1.0 / Keff                       // cb-cached, out
invKeff_reg    = 1.0 / (Keff + setup+0x54[axle])  // regularized (extra diagonal), out
```
`setup+0x54[axle]` is a per-axle **regularization / softness** term added to the longitudinal
effective mass (CFM-like). The second row repeats for the lateral/normal constraint giving a
second `Keff` (`pfVar6[3]` / its inverse `pfVar6[-1]`).

It also computes the **relative velocity at the contact** (Jv) for each row by dotting the
jacobian with `(setup+0x140.. − bodyB.vel)` — stored into `out[-1]`/`out[0]` accumulators. These
are the slip velocities used next.

### Phase B — Coupled 2×2 long/lat effective-mass matrix

After the loop, it assembles the off-diagonal coupling between the longitudinal and lateral rows
of the shared contact (the `cb+0x30 == cb+0x80` branch adds the second body's contribution):

```
M2 = [ a   b ]      a = local_3c        (long effective mass)
     [ b   d ]      d = local_e8[3]     (lat  effective mass)
                    b = local_180       (long/lat coupling)
det = a*d - b*b
if (det*det >= 0):                       // guard
    inv = 1.0/det
    Minv2 = [  a*inv   -b*inv ]          // local_160, local_15c
            [ -b*inv    d*inv ]          // local_158, local_154
```
This confirms longitudinal and lateral friction are solved as a **coupled 2×2 block**, not
independently.

### Phase C — Impulse accumulation into the bodies, loop ×2

For each axle whose gate `out+? != 0`, the previously solved per-row impulse (`out[-8..-5]` linear,
`out[-4..-1]` angular for A; `out[0..3]` for B) is applied:

```
chassisA.linImpulse[+0xc0..] += impulse * dt*Minv_lin_a * jLin
chassisA.angImpulse[+0xd0..] += impulse * dt*Iinv_a    * jAng
bodyB   .linImpulse[+0x10..] -= impulse * dt*Minv_lin_b * jLin
bodyB   .angImpulse[+0x20..] += impulse * dt*Iinv_b     * jAng
```
(`fVar30 = f*dt*Minv`, `fVar31 = f*dt*Iinv`; note the sign: reaction is subtracted on body B.)

### Phase D — Friction limits, drive solve, and FRICTION-CIRCLE clamp, loop ×2

This is the core tire model. Per axle (`pfVar9` walks the output block, `pfVar8` walks the
per-axle friction params at `cb+0x48` stride `0x14`):

**1. Longitudinal (drive) impulse:**
```
Jv = jacRow · relVel                              // fVar17, longitudinal slip velocity
if (!driveEnabled)   // *(pfVar8+1) low byte == 0
    out[axle] = 0
    lambdaLong = out[-6]           // carried
else
    lambdaLong = -Jv                              // oppose slip
    // drive push from aggregated engine torque, blended by 0.5:
    driveTarget = invKeff_reg * invKeff_lat * (drivePack*0.5 + Jv) * 0.5     // DAT_00a0f298 = 0.5
    driveMax    = (AX -0x14 field?) * invKeff_lat * |N| * dt                 // pfVar8[-4]*..*|load|*dt
    driveClamped = clamp_signed(driveTarget, ±driveMax)
    lambdaLong  = -Jv - driveClamped
```
(`|N| = |out[0xb]|` = the normal/suspension impulse magnitude for that axle.)

**2. Friction limit from tire load (slip-dependent μ):**
```
fLimit = mu0                       // pfVar8[-4]  (μ0, base friction coeff × already-scaled table)
if (slope != 0):                   // pfVar8[-3]
    slipSpeed = sqrt(lambdaLong² + lambdaLat²)          // combined slip magnitude
    fLimit    = mu0 + slipSpeed * slope                 // linear slip-friction curve
    fLimit    = min(fLimit, muMax)                       // pfVar8[-2]  (μ ceiling)
    fLimit    = max(fLimit, 0)
maxFrictionImpulse = fLimit * |N| * dt                   // × normal load × timestep
out[0x12] = out[0x13] = maxFrictionImpulse
out[0x14] = out[0x15] = 1.0 / (maxFrictionImpulse + eps) // eps = _DAT_00a0d2f4 ≈ 1.19e-7
```
So **friction impulse limit = μ(slip) · N · dt**, with μ a clamped linear function of combined
slip speed. `mu0/slope/muMax` are the per-axle friction-table entries aggregated in Part 2a — the
**rear** entries already carry `RearWheelFrictionScalar` (`vehicleData+0x740`) from setup.

**3. Friction-circle projection (coupled 2×2 solve + clamp):**
```
// solve the 2×2 block for the two friction components:
impLong = -( relVelLong*Minv2[0][0] + relVelLat*Minv2[0][1] )     // local_cc[0]
impLat  = -( relVelLong*Minv2[1][0] + relVelLat*Minv2[1][1] )     // local_2c

// normalized squared magnitude vs friction circle (per axle 0 and axle 1):
mag0 = (impLong0 * invLimit0_x)² + (impLong0 * invLimit0_y)²      // fVar3
mag1 = (impLat ...)²             + ...                            // fVar21

if (mag0 > 1.0 || mag1 > 1.0):                    // outside friction circle → scale back
    if (mag0 > 1.0): out[5] *= 1/mag0;  out[6] *= 1/mag0
    if (mag1 > 1.0): out[0xc]*= 1/mag1; out[0xd]*= 1/mag1
    hkVehicleFrictionSolver_circleProjection(out + axle*7)   // finish the projection per axle
    out[1] = ...; out[0] = ...; out[7] = ...
else:                                             // inside circle → no static friction impulse
    out[0]=out[7]=out[1]=0

out[3]  = impLong * dt2[1]        // longitudinal impulse × dt  → written back to WHEEL+0x94
out[10] = impLat  * dt2[1]
```
The `if (limit < |imp|)` scale-back by `1/mag` is the **friction circle / traction ellipse clamp**:
the combined (longitudinal, lateral) impulse vector is bounded to the disk of radius `μ·N·dt`; when
it exceeds the limit both components are scaled by the reciprocal magnitude so the direction is
preserved. If also `|something| >= cb+0xa0` the lateral outputs `out[5],out[6],out[0xc],out[0xd]`
are zeroed (a stability cutoff, likely max-slip / airborne).

**4. Final apply (second ×2 loop):** the clamped friction impulses are applied to chassis A and
body B exactly as in Phase C (`+=` linear at `+0xc0`, angular at `+0xd0`; reaction on B at
`+0x10`/`+0x20`).

## Reconstruction-ready summary (per axle, per substep)

```
invN            = 1 / axleWheelCount
drivePack       = Σ_wheels(torque_i · wheelScale_i) / axleWheelCount        // torque_i ∈ [0,1000]
suspImpulse     = suspForce · dt   (applied along contact normal, immediately)

Keff_long, Keff_lat, coupling  = J · Minv · Jᵀ  (2×2 block, + softness setup+0x54)
Jv_long, Jv_lat                = J · v_rel_at_contact

lambda_long     = -Jv_long - clamp( invKeff·(drivePack·0.5 + Jv_long)·0.5, ±(k·|N|·dt) )
slip            = sqrt(lambda_long² + lambda_lat²)
mu              = clamp( mu0 + slip·slope, 0, muMax )      // rear mu* already ×RearWheelFrictionScalar
Fmax            = mu · |N| · dt

(imp_long, imp_lat) = -Minv2 · (Jv_long, Jv_lat)          // coupled 2×2 solve
if |(imp_long,imp_lat)| > Fmax:  scale both by Fmax/|imp| // friction circle clamp

apply ±imp to chassis (lin@+0xc0, ang@+0xd0) and contact body (lin@+0x10, ang@+0x20)
write imp_long·dt → WHEEL+0x94 (longitudinal), imp_lat → WHEEL+0xa0 (lateral)
```

Constants inline: drive blend `0.5` = `DAT_00a0f298`; ang-vel weight `0.25` = `DAT_00a0f704`;
`1/x` guard eps `2^-23` = `_DAT_00a0d2f4`; `1.0`/`0.0` = `g_flOne`/`g_flZero`; torque clamp
`1000.0` = `DAT_00a0f520` (in calcWheelTorque).

## Ambiguities / open items

1. **Exact 2×2 assembly indices.** `local_3c`, `local_e8[3]`, `local_180` (a,d,b of the coupled
   matrix) and the `local_160/15c/158/154` inverse are unambiguous as *math*, but which precise
   jacobian rows feed a vs d (i.e. is index-0 the drive/longitudinal or the lateral row) is
   inferred from the writeback mapping, not proven. **Resolve:** single-step `0x6c4450` in the
   debugger with a known one-wheel-driving state and watch `out[3]` (WHEEL+0x94) vs `out[10]`.

2. **`driveMax` scale field.** The drive clamp `pfVar8[-4] * invKeff_lat * |N| * dt` reuses what
   looks like the same `mu0` slot (`cb+0x48-0x10`) as the friction limit. It may instead be a
   distinct "max drive traction" entry aggregated separately. **Resolve:** dump `cb+0x38..` for a
   front-vs-rear-drive car and compare to the setup-time friction-table build.

3. **`circleProjection` (0x6c????)** internals not decompiled here — it finalizes the projection
   when *both* axles exceed the circle. The scalar `1/mag` pre-scale is captured; the helper's
   exact renormalization (L2 vs component-wise) should be decompiled for bit-exactness. **Resolve:**
   decompile `hkVehicleFrictionSolver_circleProjection` (the call site passes `param_4 + axle*7`).

4. **Softness term composition.** `Keff + setup+0x54[axle]` is clearly a regularization add, but
   whether `setup+0x44`/`+0x94` (used in the `local_e8[1]`/`local_44` pre-terms) are additional
   softness or a warm-start bias is unconfirmed. **Resolve:** inspect the friction-setup builder
   that populates `fw+0x1fc`.

5. **`cb+0xa0` lateral cutoff threshold.** The condition `cb+0xa0 <= |local_f0|` that zeroes the
   lateral outputs is a max-slip or airborne guard; the meaning of `local_f0` (a normal-direction
   term) is inferred. **Resolve:** watch `cb+0xa0` and `local_f0` across a normal→sliding
   transition.

None of these affect the *shape* of the model (coupled 2×2 friction with μ(slip)·N·dt circle clamp,
drive impulse from averaged per-wheel engine torque); they affect exact field bindings only.
