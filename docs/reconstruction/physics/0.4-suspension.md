# Phase 0.4 — Havok Vehicle Suspension (spring + damper) & Wheel Struct Layout

Reverse-engineered from `autoassault.exe` (image base 0x400000), Ghidra project `AA-decode`.
Havok 2.3 `hkVehicleFramework` / `hkDefaultSuspension`. Read-only analysis.

The AA client is a thin `VehicleAction` layer over the stock Havok vehicle SDK. The
suspension is stock Havok: `hkDefaultSuspension`. The per-tick force pipeline is:

```
VehicleAction::tickSubsystems (0x636a60)
  -> hkVehicleFramework::preUpdate      (0x64cf20)   raycast + wheel geometry, writes wheel+0xac/0xb0/0xb4
  -> [7 component updates, vtbl+0x14]
       -> hkDefaultSuspension::update   (0x64de50)   << SPRING+DAMPER FORCE, writes susp+0x34[i]
  -> hkVehicleFramework::postTickApplyForces (0x64bc70) applies susp+0x34[i] to chassis as impulse
```

---

## 1. Suspension force formula — `hkDefaultSuspension::update` @ 0x64de50

Definitive, bit-exact source:

```c
// param_1 = suspension component (framework+0x28)
wheels   = *(comp+8 [framework] + 0xc);              // hkDefaultWheels container
chassisS = *(*(comp+8 +0x30 [chassis]) +0x3c) +0x2c; // chassis rigid-body scalar @ RB+0x2c
gScale   = (chassisS == 0) ? 0 : 1.0f / chassisS;    // global force normalizer

for (i = 0; i < comp+0x40 [wheelCount]; i++) {
    wheel = *(wheels+0x80) + i*0xC0;                 // wheel struct, stride 0xC0
    if (wheel+0x80 == 0) {                           // not in contact
        susp+0x34[i] = 0;
    } else {
        // pick damping coefficient by closing-speed sign
        dampArr = (wheel+0xB4 >= 0) ? comp+0x5C      // EXTENSION (rebound) damping
                                    : comp+0x50;     // COMPRESSION damping
        susp+0x34[i] =
            ( (comp+0x28[i] - wheel+0xB0)            // (restLength - currentLength) = compression
              * comp+0x44[i]                         // * spring strength
              * wheel+0xAC                           // * suspension scaling factor (1/len)
              - dampArr[i] * wheel+0xB4 )            // - dampingCoef * closingSpeed
            * gScale;                                // * global normalizer
    }
}
```

Restated:

```
compression   = restLength[i] - currentLength[i]           // = comp+0x28[i] - wheel+0xB0
springForce   = compression * strength[i] * scalingFactor  // scalingFactor = wheel+0xAC
closingSpeed  = wheel+0xB4                                  // >0 extending, <0 compressing
dampCoef      = (closingSpeed >= 0) ? extensionDamp[i]      // comp+0x5C  (rebound)
                                    : compressionDamp[i]    // comp+0x50
damperForce   = dampCoef * closingSpeed
suspForce[i]  = (springForce - damperForce) * (1 / chassisRB[+0x2c])   // 0 if airborne
```

**Compression vs extension damping selection (answers the key question):**
- `closingSpeed (wheel+0xB4) < 0` → suspension is **compressing** → use **Compression** coefficient (comp+0x50).
- `closingSpeed (wheel+0xB4) >= 0` → suspension is **extending/rebounding** → use **Extension** coefficient (comp+0x5C).

The spring term uses `wheel+0xAC` (a per-wheel `1/length` scaling factor set in preUpdate; on
a full raycast it is `1/maxSuspensionLength`, on a shorter hit `-1/hitDistance`). The whole
per-wheel result is scaled by `gScale = 1/chassisRB[+0x2c]` (chassis rigid-body scalar, same
normalizer `postTickApplyForces` uses — empirically the chassis mass/inverse-mass normalizer;
flag for live confirmation).

## 2. How the force reaches the chassis — `postTickApplyForces` @ 0x64bc70

```
per wheel i (in contact):
    F        = susp+0x34[i]                       // scalar from update() above
    impulse  = F * dt * (wheel+0x30 .. +0x3c)     // along contact normal (4-vec @ wheel+0x30)
    chassisRigidBody->vtbl[0x60]( impulse, wheel ) // applyPointImpulse at wheel contact point (wheel+0x20)
```

So the suspension force acts along the **contact normal** (`wheel+0x30`), applied at the wheel
**contact/hardpoint** (`wheel+0x20`). `dt = param_2[0]`.

## 3. How wheel compression / closingSpeed are produced — `preUpdate` @ 0x64cf20

Per wheel, after the wheel raycast/collide (`param_1->vtbl[0x20]` fills contact @ `&fStack_a8`):

- **No contact** (`iStack_88 == 0`): `wheel+0x80 = 0` (airborne flag), `wheel+0xB0 = restLen`,
  `wheel+0xAC = 1.0`, `wheel+0xB4 = 0`, contact normal `wheel+0x30 = -(wheel+0x50 downAxis)`.
- **Contact**: writes contact normal to `wheel+0x30..0x3c`, contact body to `wheel+0xA4`, sets
  `wheel+0x80 = 1`. Current length `wheel+0xB0 = (restLen + hitLen)*shockScale - restLen`
  (shockScale `fStack_94` defaults 1.0). Then:
  - if `hitDist >= -maxSuspLen (framework+0x304)`: `wheel+0xB4 = 0`, `wheel+0xAC = 1/maxSuspLen`.
  - else: `wheel+0xAC = -1/hitDist`, `wheel+0xB4 = (-1/hitDist) * dot(contactNormal, hardpoint-contactPoint)`
    (the geometric closing term the damper multiplies).
- Also computes wheel spin: `wheel+0x8C = (wheel+0x9C + longContactVel)/wheelRadius`,
  `wheel+0x90 += dt*wheel+0x8C`.

`maxSuspLen` lives at `framework+0x304` (`param_1[0xC1]`), per-vehicle runtime value.

## 4. Wheel struct layout (stride 0xC0, base `hkDefaultWheels+0x80`, array count `wheels+0xC`)

All vectors are 4-float (x,y,z,w). Confirmed from preUpdate/postTick/update/calcWheelTorque.

| Offset | Type | Meaning |
|--------|------|---------|
| +0x00 | vec4 | wheel hardpoint base (chassis space) — rest pos source |
| +0x10 | vec4 | (hardpoint working copy) |
| +0x20 | vec4 | **contact point** (world). +0x20/+0x28 = torqueCurve2D args in calcWheelTorque |
| +0x30 | vec4 | **contact normal** (world) = suspension force direction; airborne = -downAxis |
| +0x40 | vec4 | chassis basis row (copied from chassisRB+0x30) — spin axis frame |
| +0x50 | vec4 | suspension **down axis** / hardpoint direction |
| +0x70 | vec4 | normalized steer/roll basis vector (written near end of preUpdate) |
| +0x74/78/7c | float | components of the +0x70 vector |
| +0x80 | byte | **in-contact flag** (1 = grounded, 0 = airborne). Gates force in update()/calcWheelTorque |
| +0x88 | float | **drive-torque axle scale** (calcWheelTorque out × this ÷ axleWheelCount in postTick) |
| +0x8C | float | wheel spin angular velocity (rad/s) |
| +0x90 | float | accumulated wheel spin angle |
| +0x94..0xA0 | 4×float | friction-solver writeback (postTick writes +0x94/98/9C/A0) |
| +0x9C | float | longitudinal contact velocity (spin-calc input; overwritten by writeback) |
| +0xA4 | ptr | contact rigid body |
| +0xA8 | byte | contact body "fixed/collidable" flag |
| +0xAC | float | **suspension scaling factor** (1/maxSuspLen, or -1/hitDist) — spring multiplier |
| +0xB0 | float | **suspension current length** (compression scanned by applyAction "most compressed") |
| +0xB4 | float | **suspension closing speed** (damper velocity; <0 compress, >=0 extend) |

(applyAction @0x598650 scans `wheel+0xB0` across wheels to find the most-compressed for its
airborne/landing logic; calcWheelTorque @0x598040 gates on `wheel+0x80` and reads +0x20/+0x28.)

## 5. `hkDefaultSuspension` component layout (framework+0x28)

vtable `PTR_FUN_009e4c00`. ctor 0x64e510, array-copy 0x64df10. Each array is an `hkArray`
(ptr, size, capacity|0x80000000).

| Offset | Meaning | Source (VehicleSpecific) |
|--------|---------|--------------------------|
| +0x08 | back-ptr to framework | — |
| +0x28 | ptr: **rest length[]** | SuspensionLength (Front/Rear) |
| +0x34 | ptr: **output force[]** (read by postTick) | — (computed) |
| +0x40 | int: wheel count | — |
| +0x44 (0x44/48/4C) | ptr: **spring strength[]** | SuspensionStrength (Front/Rear) |
| +0x50 (0x50/54/58) | ptr: **compression damping[]** | SuspensionDampeningCoefficientCompression (Front/Rear) |
| +0x5C (0x5C/60/64) | ptr: **extension damping[]** | SuspensionDampeningCoefficientExtension (Front/Rear) |

Also holds hardpoint (ShockAttachPoints) and down-axis vec4 arrays set by the base ctor
(FUN_0065e070) from descriptor slots +0x00 / +0x0C.

## 6. VehicleSpecific → Havok mapping (setup chain)

`Vehicle::buildHavokVehicleFramework` @ 0x5fd390 →
`Vehicle_BuildSuspensionDescriptor` @ 0x5fcff0 (builds stack descriptor) →
`hkDefaultSuspension_ctor` @ 0x64e510 + `FUN_0064df10` @ 0x64df10 (copies descriptor→component).

`BuildSuspensionDescriptor` reads per-wheel FrontRear pairs from
`vehicleData(+0x3c)` at consecutive offsets (Front used for wheels `< NumFrontWheels`
(byte @ +0x4CC), Rear otherwise):

| vehicleData off | Descriptor slot | Component off | VehicleSpecific field (FrontRear) |
|-----------------|-----------------|---------------|-----------------------------------|
| +0x55C / +0x560 | param_3[6] (+0x18) | +0x28 (restLength) | **SuspensionLength** {Front;Rear} |
| +0x564 / +0x568 | param_3[9] (+0x24) | +0x44 (strength)   | **SuspensionStrength** {Front;Rear} |
| +0x56C / +0x570 | param_3[0xC] (+0x30) | +0x50 (compDamp) | **SuspensionDampeningCoefficientCompression** {Front;Rear} |
| +0x574 / +0x578 | param_3[0xF] (+0x3C) | +0x5C (extDamp)  | **SuspensionDampeningCoefficientExtension** {Front;Rear} |

`FUN_0064df10` copy confirms: comp+0x44 ← desc+0x24, comp+0x50 ← desc+0x30, comp+0x5C ← desc+0x3C,
count ← desc+0x28. comp+0x28 (restLength) and the hardpoint/axis vec4s are set by the base ctor
from desc+0x18 / +0x00 / +0x0C.

- **ShockAttachPoints** (Vector3[]) → descriptor param_3[0] hardpoint vec4 array
  (from `vehicleData+0x514`, stride 0xC). Suspension direction fixed to `(0, -1, 0)` (desc param_3[3]).
- **WheelHardPoints / WheelRadius** feed the separate `hkDefaultWheels` descriptor (FUN_005fcce0),
  not the suspension component. WheelRadius appears in preUpdate spin calc (`/wheelRadius`).
- **ShockScale / ShockEffectThreshold** (float[]) — not consumed by the physics suspension force;
  `preUpdate` uses a shock scale (`fStack_94`) that defaults to 1.0 (visual/graphics shock path).

The DB→VehicleSpecific field names (`VehicleDb_LoadCloneBase` @ 0x7f2be0, string keys at
0xa93634/0xa9366c/0xa935a0/0xa93540/0xa934e0/0xa93480/0xa9369c/0xa93634): `rlSuspensionLengthFront/Rear`,
`rlSuspensionStrengthFront/Rear`, `rlSuspensionDampeningCoefficientCompressionFront/Rear`,
`rlSuspensionDampeningCoefficientExtensionFront/Rear`.

## 7. Constants (addr = value)

| Symbol | Addr | Value | Use |
|--------|------|-------|-----|
| DAT_00aaa668 | 0x00aaa668 | **-1.0** | suspension down-axis Y in descriptor; preUpdate `wheel+0xAC = -1/dist` numerator |
| g_flOne | (1.0) | 1.0 | gScale numerator; default scaling factor / shockScale |
| DAT_00a0f298 | 0x00a0f298 | 0.5 | (rear-traction cut in calcWheelTorque / friction basis — not suspension force) |
| maxSuspLen | framework+0x304 | runtime | preUpdate scaling-factor threshold `1/maxSuspLen` |
| chassisRB+0x2c | runtime | mass/normalizer | `gScale = 1/RB[+0x2c]` global force scale (live-confirm) |

Note: the only tunable per-vehicle DATA is the four FrontRear pairs (Length/Strength/CompDamp/ExtDamp)
plus ShockAttachPoints; everything else in the force formula is fixed code (down axis, gScale, sign rule).

## 8. C# port pseudocode

```csharp
// per wheel, per tick
if (!wheel.InContact) { wheel.SuspForce = 0f; continue; }

float compression  = restLength[i] - wheel.CurrentLength;      // wheel+0x28 desc, wheel+0xB0
float spring       = compression * strength[i] * wheel.ScalingFactor; // wheel+0xAC
float dampCoef     = wheel.ClosingSpeed >= 0f ? extensionDamp[i]      // rebound
                                              : compressionDamp[i];   // compress
float damper       = dampCoef * wheel.ClosingSpeed;            // wheel+0xB4
wheel.SuspForce    = (spring - damper) * gScale;               // gScale = 1/chassisMassScalar

// apply (postTick):
Vector3 impulse = wheel.SuspForce * dt * wheel.ContactNormal;  // wheel+0x30
chassis.ApplyPointImpulse(impulse, wheel.ContactPoint);        // wheel+0x20
```
