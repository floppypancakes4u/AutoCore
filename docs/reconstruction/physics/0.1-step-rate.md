# Phase 0.1 — Physics step rate / dt (RESOLVED)

## Answer
The client does **not** use a single fixed dt. The Havok vehicle sim runs a **frame-delta-driven
sub-stepping accumulator capped at ~30 Hz**:

```
frameDt    = min(frameDt, 0.1)                 // clamp: physics floor = 10 fps
N          = floor(frameDt * 29.9999998) + 1   // sub-step count
substep_dt = frameDt / N                        // dt handed to applyAction (param_2[0])
```

Behaviour:
- 60 fps (frameDt 0.01667) → N=1 → **dt = 1/60 ≈ 0.0166667**
- 30 fps (frameDt 0.03333) → N=1 → **dt = 1/30 ≈ 0.0333333**
- < 30 fps → splits into N equal sub-steps so **no sub-step ever exceeds 1/30 s**.

**Invariant for the server port:** variable dt, **max sub-step = 1/30 s**; reproduce with
`N = floor(frameDt*30)+1`, `substep_dt = frameDt/N`, `frameDt` clamped ≤ 0.1 s. A clean fixed
1/60 s reproduces the common 60 fps client path exactly; 1/30 s is the hard upper bound.

## Call chain (tick site → applyAction)
```
CVOGSectorMap::StepTo        0x004d6c80   builds substep_dt = frameDt/N, loops N times
  └ FUN_00561910             0x00561910   island sim step ("LtSimulate"); stores dt at island+0x150
      └ FUN_00629d90                      island integrate / applyActions (arg island+0x140 stepInfo)
          └ VehicleAction::applyAction  0x00598650   param_2[0] = substep_dt
```
- `applyAction` = slot **+0x14** of VehicleAction vtable **`0x009d54c4`** (set by ctor `FUN_00597e90`).
- VehicleAction registered into world/island action list: `Vehicle_createVehicleAction @ 0x4fb660`
  → `FUN_0055fe50` → `FUN_006292a0` (world+0x4c).
- `tickSubsystems @ 0x636a60` → `hkVehicleFramework_preUpdate @ 0x64cf20` integrates wheel spin
  `angle += dt * spinSpeed` (wheel+0x90), then friction solver.

## Server-port note
Model the accumulator on the client's rule (N = floor(dt*30)+1) rather than a plain fixed Hz.
`ServerConfig.SubstepHz` currently defaults 60; Phase 4 accumulator should implement the
`frameDt/N` sub-step split with the 1/30 s cap. Consider exposing the cap (30) instead of a nominal.
