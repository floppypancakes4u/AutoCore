# Phase 0.5 — Havok Vehicle Wheel-Collide (wheel→ground cast) & Per-Wheel Friction Source

Reverse-engineered from `autoassault.exe` (Ghidra project `AA-decode`, image base `0x400000`, Havok 2.3).
Read-only RE. Wheel stride `0xC0`, base `wheels+0x80` (`wheels = framework[3] = *(fw+0xc)`).

## 2. Cast method — Havok collision cast, NOT terrain-height

`wheelCollide` (`0x64bbd0`) copies the wheel's stored ray (wheel+0x00..0x0c = start hardpoint,
wheel+0x10..0x1c = end, 8 floats total), sets `this = *(fw+0x1f8)` (the vehicle's **TtPhantom**),
and tail-calls `TtPhantom::castRay` (`0x580ed0`).

`TtPhantom::castRay` (string `"TtPhantom::castRay"` @ `0x9d4574`) iterates the phantom's
broadphase **overlap list** (`phantom+0x80` = collidable array, `phantom+0x84` = count),
transforms the world ray start/end into each body's local frame via that body's transform
(`body+0x20..0x48` = 3×3 rot, `+0x50..0x58` = translation), then calls each shape's
`hkShape::castRay` **virtual slot +0x20** with the local ray + result. On hit it stores the hit
collidable at `result+0x20`.

**Conclusion:** the wheel cast is a **per-shape Havok collision raycast against bodies overlapping
the vehicle phantom** — a different path from `CVOGMap::CastTerrainHeight` @ `0x4cfe60` (which
casts one ray via `FUN_0055e530` against map collision for creature placement). The wheel cast
also hits *movable* rigid bodies (result+0x20 body, checked for `body+0x18==1`), which the terrain
cast does not. Related strings: `TthkShapeCollection::castRay` @ `0xa0e2d4`, `hkRaycastVehicle`
@ `0x9e5214` (RTTI/reflection only).

### Contact result struct (written by castRay, read in preUpdate at `&fStack_a8`)
| Off | Field |
|---|---|
| +0x00..0x0c | contact **normal** (world, 4 floats) |
| +0x14 | hit **fraction** along ray (default `1.0` = miss) |
| +0x20 | hit **collidable/body** pointer (0 = miss) |

## 3. Wheel struct writes (preUpdate `0x64cf20`, contact branch)

`pfVar7 = wheels+0x80 + i*0xC0`. `fraction = result+0x14`; `radius = (*(wheels+0x10))[i]`;
`suspLen = (*(fw[3]+0x28))[i]` (suspension rest/max length).

| Wheel off | Write | Meaning |
|---|---|---|
| **+0x30..0x3c** | `= result normal` | contact normal basis (used by suspension impulse in postTick) |
| **+0x80** (byte) | `= 1` (hit) / `0` (miss) | **in-contact flag** |
| **+0xa4** | `= result+0x20` | hit body pointer (0 on miss) |
| +0xa8 (byte) | body movable flag (`body+0x40`) | |
| +0xac | `= 1/suspLen` (miss: `1.0`) | normalization |
| **+0xb0** | **`(radius+suspLen)*fraction − radius`** (miss: `suspLen`) | **compression length** |
| +0xb4 | `= (−1.0/den)·dot(normal, contactPt−hardpoint)` (miss `0`) | suspension-rate term |

**Compression formula:** `wheel+0xb0 = (wheelRadius + suspensionRestLength) · hitFraction − wheelRadius`.
Fraction 1.0 (miss/untouched) ⇒ compression = full `suspLen` (extended). Spin integrated after:
`wheel+0x8c = spinAccel`, `wheel+0x90 += dt·spin` (grounded only, else `+0x8c=0`).

## 4. Per-wheel friction coefficient source

`FUN_004f5550(idx)` disasm:
```
mov ecx,[ecx+0x258]      ; this = *(component+0x258)  → the wheelset/VehicleSpecific data
jmp 0x5a6f20             ; return *(this + 0xb4 + idx*4)
```
`FUN_005a6f20` is a 6-case switch returning `this+0xb4[idx]` (idx 0..5), default `DAT_00aaa668=-1.0`.
`FUN_004f5560` returns wheelCount = byte `this+0xb0`. So the wheelset object holds:
`+0xb0` wheelCount, `+0xb4..0xc8` = **per-wheel base friction array** (μ_base, up to 6).

`calcWheelTorque` (`0x598040`) calls `FUN_004f5550(i)` directly → uses the **un-scaled** μ_base
(then × upright × torqueCurve, × low-speed boost, clamp [0,1000]).

### Friction-table setup + RearWheelFrictionScalar — `FUN_005fcce0` @ `0x5fcce0`
Builds the solver's parallel per-wheel arrays (`param_3` block; these become the μ0/μmax params
aggregated per-axle in postTick — see 0.3). Per wheel `i`:
```
firstRear = *(vehicleData+0x3c → +0x4cc)          // first rear-wheel index
isFront   = (i < firstRear)                        // → stored as uint at (param_3+0x04)[i]
mu0       = FUN_004f5550(i)                         // base friction  → (param_3+0x28)[i]
if (i >= firstRear)                                 // REAR wheels:
    mu0 *= *(vehicleData + 0x740)                   //   × RearWheelFrictionScalar
muMax     = mu0 * 1.5   (DAT_00aaa68c)              // → (param_3+0x40)[i]
(param_3+0x34)[i] = g_flMsToSeconds_Inferred        // friction slope field
(param_3+0x4c)[i] = 0.01  (DAT_00a0f718)
(param_3+0x58)[i] = 15.0  (DAT_00aaa7a4)            // low-speed boost threshold
(param_3+0x10)[i] = vehicleData+0x600[i]            // (WheelHardPoints-related vec)
(param_3+0x1c)[i] = vehicleData+0x618[i]
vehicleData = *(*(*(param_1+4)+4)+0xac+param_1) + 0x3c
```
**RearWheelFrictionScalar = `vehicleData+0x740`** (VehicleSpecific). It scales the solver's rear
μ0 (and thus μmax) at setup; the rear entries carried into postTick already include it — matching
0.3's note. It is **not** the `0.5` in calcWheelTorque (`DAT_00a0f298`, a separate handbrake cut).

## 6. Ambiguities

1. **`FUN_004f5550` `this`** — reads `*(component+0x258)+0xb4`. The `+0x258` container (VehicleAction
   vs component) is inferred from the tail-call ECX; the wheelset object layout (`+0xb0` count,
   `+0xb4..` μ array ≤6) is confirmed by the switch. Which higher struct owns `+0x258` not fully
   pinned. Resolve: watch ECX at `0x4f5550` for a known car.
2. **`g_flMsToSeconds_Inferred`** stored to `(param_3+0x34)` is the friction **slope** field
   (solver `pfVar8[-3]`); its literal value/name not re-read here.
3. **wheel+0xb4 rate term** (`−1.0/den · dot(normal, Δpos)`) — `den` = `fw+0x304` (`param_1[0xc1]`,
   a max-suspension clamp); exact physical meaning (damper velocity) inferred, not proven.
4. **castRay hit fraction vs `fStack_94`** — preUpdate seeds `fStack_94=1.0` and reads it back as
   the compression fraction; confirmed the shape castRay overwrites it on hit, but the exact result
   field index (+0x14) is inferred from the local layout, not disassembled inside `0x51e900`.
